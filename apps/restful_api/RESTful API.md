## 角色
你是一名**极其严格的后端开发专家**，项目使用 **Dart** 语言。你的任务是：**从零开始（或基于现有项目）实现一套完整的 RESTful API**，严格满足以下所有需求。不允许有任何模糊、遗漏或简化。

## 任务
1. 实现以下核心 API 模块：用户、视频、商品、购物车、订单、评论。
2. 每个模块必须提供标准的 CRUD 能力（除非特别说明）。
3. 必须支持：
   - 内容列表分页（`page`、`pageSize` 或 `offset`、`limit`）
   - 商品查询（至少支持按关键词或分类检索）
   - 购物车增删改查（添加商品、修改数量、删除商品、查询购物车）
   - 订单创建（从购物车下单并生成订单）
4. 输出可运行的 Dart 代码（使用 **Dart Frog** 框架），并提供清晰的 API 文档（端点、方法、请求/响应示例）。
5. 所有数据存储使用**内存模拟**（例如 `List<Map>`）或 JSON 文件，不需要真实数据库。

## 需求清单（绝对标准，每一条都必须实现）

### 一、用户 API
| 端点                  | 方法 | 功能         | 要求                                                         |
| --------------------- | ---- | ------------ | ------------------------------------------------------------ |
| `/api/users/register` | POST | 用户注册     | 接收 `username`、`password`、`nickname`，返回 `userId` 和模拟 token |
| `/api/users/login`    | POST | 用户登录     | 接收 `username`、`password`，返回 token 和用户信息           |
| `/api/users/profile`  | GET  | 获取个人信息 | 需要 token（请求头 `Authorization: Bearer <token>`），返回用户资料 |
| `/api/users/profile`  | PUT  | 更新个人信息 | 可更新昵称、头像等，返回更新后的用户信息                     |

### 二、视频 API
| 端点                             | 方法 | 功能                 | 要求                                                         |
| -------------------------------- | ---- | -------------------- | ------------------------------------------------------------ |
| `/api/videos`                    | GET  | 获取视频列表         | 支持分页参数 `page`、`pageSize`，返回视频列表（视频 id、标题、封面、播放地址、点赞数、评论数） |
| `/api/videos/{videoId}`          | GET  | 获取视频详情         | 返回完整视频信息及关联商品列表                               |
| `/api/videos/{videoId}/like`     | POST | 点赞视频             | 需要 token，点赞状态切换                                     |
| `/api/videos/{videoId}/comments` | GET  | 获取视频下的评论列表 | 支持分页                                                     |

### 三、商品 API
| 端点                        | 方法 | 功能         | 要求                                                         |
| --------------------------- | ---- | ------------ | ------------------------------------------------------------ |
| `/api/products`             | GET  | 获取商品列表 | 支持分页，支持查询参数 `keyword`（标题搜索）、`categoryId` 分类筛选 |
| `/api/products/{productId}` | GET  | 获取商品详情 | 返回商品图、标题、价格、优惠信息、库存状态、规格列表         |

### 四、购物车 API
| 端点                       | 方法   | 功能                 | 要求                                                         |
| -------------------------- | ------ | -------------------- | ------------------------------------------------------------ |
| `/api/cart`                | GET    | 获取当前用户的购物车 | 返回购物车中所有商品项（商品信息、数量、选中状态）           |
| `/api/cart/items`          | POST   | 添加商品到购物车     | 接收 `productId`、`quantity`、`spec`（可选），返回更新后的购物车 |
| `/api/cart/items/{itemId}` | PUT    | 修改购物车项数量     | 接收 `quantity`（≥1），返回更新后的购物车                    |
| `/api/cart/items/{itemId}` | DELETE | 删除购物车项         | 返回更新后的购物车                                           |
| `/api/cart/items/selected` | PUT    | 批量勾选/取消勾选    | 接收 `itemIds` 列表和 `selected` 布尔值，返回更新后的购物车  |

### 五、订单 API
| 端点                        | 方法 | 功能         | 要求                                                         |
| --------------------------- | ---- | ------------ | ------------------------------------------------------------ |
| `/api/orders`               | POST | 创建订单     | 接收 `addressId`、`cartItemIds`（购物车勾选的项），返回订单对象（订单号、总金额、实付金额、状态为“待支付”） |
| `/api/orders`               | GET  | 获取订单列表 | 支持分页，支持查询参数 `status`（pending_payment, pending_delivery, completed）用于筛选 |
| `/api/orders/{orderId}`     | GET  | 获取订单详情 | 返回订单完整信息（商品列表、地址、优惠明细、实付金额）       |
| `/api/orders/{orderId}/pay` | POST | 模拟支付     | 将订单状态从“待支付”改为“已支付/待发货”，返回支付结果        |

### 六、评论 API
| 端点                        | 方法   | 功能         | 要求                                                         |
| --------------------------- | ------ | ------------ | ------------------------------------------------------------ |
| `/api/comments`             | POST   | 发布评论     | 需要 token，接收 `targetType`（video/product）、`targetId`、`content`，返回评论对象 |
| `/api/comments/{commentId}` | DELETE | 删除评论     | 需要 token，仅允许删除自己的评论                             |
| `/api/comments`             | GET    | 获取评论列表 | 按 `targetType` 和 `targetId` 筛选，支持分页                 |

## 技术约束
- 使用 **Dart Frog** 框架（轻量、快速）。如果项目中没有，请提供完整的 `pubspec.yaml` 和启动命令。
- 使用 **模拟数据存储**：在内存中维护 `List` / `Map`（例如 `Map<String, List<Map>>`），每次启动时初始化少量示例数据（至少 2 个用户、5 个商品、3 个视频、若干评论和购物车数据）。
- 所有 API 必须返回规范的 JSON 响应，格式如下：
  ```json
  {
    "code": 200,
    "message": "success",
    "data": { ... }   // 或数组
  }

错误时返回相应的 HTTP 状态码（400、401、404 等）及错误信息。

- 需要实现简单的 token 验证中间件：从请求头提取 `Authorization`，校验 token 有效性（模拟：只要 token 不为空且对应存在的用户即通过，可暂不实现过期）。
