## 角色
你是一名**极端严格的实时通信开发专家**，项目使用 **Dart** 语言（Flutter 前端 + Dart 后端）。你的任务是：**从零开始（或基于现有项目）完整实现直播间实时通信功能**，所有需求必须 100% 满足，不允许有任何偏差、替代方案或降级实现。

## 核心要求
使用 **WebSocket** 实现双向实时通信，完成以下功能：
- 实时接收/发送直播间评论
- 实时接收弹幕（可由评论转换而来或独立发送）
- 实时更新在线人数（多人连接时同步人数变化）
- 实时推送当前讲解的商品信息（后台或主播可切换讲解商品，所有客户端自动更新）

## 需求清单（逐字逐句满足）

### 1. WebSocket 连接与房间管理
- 前端可以连接到 WebSocket 服务器，并携带 `roomId`（直播间 ID）参数。
- 服务器维护多个直播间的连接池，同一房间内的消息只广播给该房间的客户端。
- 当客户端断开连接时，服务器从房间中移除该连接，并更新该房间的在线人数。

### 2. 实时评论
- 客户端可以发送评论消息（包含 `userId`、`nickname`、`content`、`timestamp`）。
- 服务器收到评论后，**立即广播**给该房间内的所有客户端。
- 客户端收到评论后，显示在**评论列表**中（最新评论在上方或下方，需明确）。
- 必须支持至少 3 条预置示例评论展示（连接后立即展示）。

### 3. 弹幕功能
- 弹幕与评论可以共用同一种消息类型，但 UI 上需要以弹幕形式从右向左滚动飘过。
- 或者独立发送弹幕消息（满足交互即可）。要求：当客户端发送一条消息时，可以选择同时展示为评论（在评论列表）和弹幕（飘过）；或者至少有模拟数据展示弹幕效果。
- **必须**有弹幕 UI 组件（覆盖在视频上方），且接收到的实时消息能在弹幕区域飘过。

### 4. 在线人数实时更新
- 当有客户端进入或离开直播间时，服务器更新该房间的在线人数，并**广播给房间内所有客户端**。
- 客户端收到在线人数更新后，立即更新界面显示（例如“1.2k 人在看”）。
- 初始连接时，服务器应返回当前房间的在线人数。
- 在线人数不需要持久化，基于当前 WebSocket 连接数即可。

### 5. 当前讲解商品实时推送
- 服务器需要有一个接口或模拟的主播角色，可以**切换正在讲解的商品**（例如通过 HTTP 请求或专门的 WebSocket 消息）。
- 当讲解商品发生变化时，服务器向该房间所有客户端广播一条“讲解商品切换”消息，包含商品 ID、名称、图片等基本字段。
- 客户端收到后，更新 UI 上的“正在讲解”标签以及商品卡片高亮状态。
- 为了演示，需要提供一种模拟触发讲解商品切换的方式（例如：另一个 WebSocket 客户端发送特定指令，或提供一个简单的 HTTP Admin 端点来切换）。

### 6. 客户端实时接收并更新 UI
- Flutter 客户端必须使用 `web_socket_channel` 包管理 WebSocket 连接。
- 需要提供完整的 `LiveRoomPage` 页面，包含：
  - 视频播放区域（可用占位图）
  - 评论列表（显示头像、昵称、评论内容）
  - 弹幕叠加层（覆盖在视频上）
  - 在线人数显示
  - 当前讲解商品卡片（突出显示）
  - 发送消息的输入框（可同时作为评论/弹幕输入）
- 所有实时数据的变化必须立即反映在界面上，无需手动刷新。

## 技术实现约束
- **后端**：使用 Dart 框架（推荐 **Dart Frog** + `dart_frog_web_socket` 或 **shelf_web_socket**）。
- **前端**：Flutter，使用 `web_socket_channel: ^2.4.0`。
- 数据格式：使用 JSON 格式定义消息类型（例如 `type` 字段区分：`comment`、`online_count`、`explaining_product` 等）。
- 所有数据存储在内存中（房间列表、连接池），无需数据库。
- 必须提供完整的启动说明（后端运行命令、前端运行命令）。
