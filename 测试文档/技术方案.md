## 一、需求概述

### 1.1 术语解释

| 术语     | 解释                                                         |
| :------- | :----------------------------------------------------------- |
| Feed     | 短视频信息流，用户垂直滑动浏览的全屏视频列表页面             |
| 商品卡   | 短视频/直播中浮层展示的商品信息卡片，用户点击可查看详情或直接加购 |
| 讲解商品 | 直播间当前正在讲解的商品，由主播推送切换，通过 WebSocket 同步显示给所有观众 |
| 弹幕     | 直播间内从右向左飘过的实时消息，支持文本消息与系统通知       |

### 1.2 需求整理

本方案覆盖以下功能模块的实现：

- **内容流浏览**：短视频 Feed 流与直播列表的上下滑动切换，每个内容页展示视频画面、主播/作者信息、点赞、评论、分享、商品入口等基础元素，支持视频播放、暂停、静音、进度展示
- **RESTful API**：提供用户、视频、商品、购物车、订单、评论等核心业务 API，支持内容列表分页、商品查询、购物车增删改查、订单创建
- **实时通信**：使用 WebSocket（Socket.IO）实现直播间弹幕、在线人数、当前讲解商品等实时推送，用户端实时接收并更新直播间状态
- **数据持久化**：服务端通过 API 持久化业务数据（商品、视频、购物车、订单），客户端通过 SharedPreferences + Hive 缓存用户状态、收藏、浏览历史

## 七、核心实现方案

### 7.1 内容流浏览

#### 7.1.1 短视频 Feed 流

采用 `PageView.builder` 实现垂直全屏滑动，通过 `FeedNotifier` 管理视频列表状态和当前播放索引。

dart

```
// FeedNotifier 核心状态管理（lib/provider/feed_provider.dart）
final class FeedState {
  final List<VideoModel> videos;
  final int currentIndex;
  final bool isLoading;
  final bool isLoadingMore;
  final bool hasMore;
  final int page;
  final FeedTab tab;           // recommend | follow
  final String? errorMessage;
}

final class FeedNotifier extends StateNotifier<FeedState> {
  // 双 Tab 切换
  Future<void> switchTab(FeedTab tab) async {
    state = state.copyWith(tab: tab, videos: [], currentIndex: 0);
    await loadVideos();
  }

  // 无限滚动：当前索引 ≥ 总长度-3 时自动加载下一页
  void setCurrentIndex(int index) {
    state = state.copyWith(currentIndex: index);
    _preloadAround(index);
    if (index >= state.videos.length - 3 && state.hasMore) {
      loadMore();
    }
  }

  // 播放器预加载：预加载当前视频前后各 1 个
  void _preloadAround(int index) {
    for (int i = index - 1; i <= index + 1; i++) {
      if (i >= 0 && i < videos.length && i != index) {
        pool.preload(videos[i].id, videos[i].videoUrl);
      }
    }
  }
}
```

#### 7.1.2 播放器池化

`PlayerPool` 管理 3 个 `VideoPlayerController` 实例，基于引用计数淘汰空闲控制器。

// lib/services/player_pool.dart
final class PlayerPool {
  static const int _poolSize = 3;
  final _controllers = <String, _PooledController>{};

  // 获取或创建控制器
  VideoPlayerController? getController(String videoId, String url) { ... }

  // 预加载视频（不创建控制器，仅缓存 URL 映射）
  void preload(String videoId, String url) { ... }

  // 淘汰策略：释放引用计数最低的控制器
  void _evictIfNeeded() {
    if (_controllers.length >= _poolSize) {
      final toEvict = _controllers.entries
        ..sort((a, b) => a.value.refCount.compareTo(b.value.refCount));
      toEvict.first.value.controller.dispose();
      _controllers.remove(toEvict.first.key);
    }
  }
}

### 7.3 实时通信

#### 7.3.1 WebSocket 服务实现

```
// lib/services/websocket_service.dart
final class WebSocketService {
  io.Socket? _socket;
  int _reconnectAttempts = 0;
  static const int _maxReconnectAttempts = 5;

  // 广播流
  final _stateController = StreamController<WebSocketState>.broadcast();
  final _eventController = StreamController<Map<String, dynamic>>.broadcast();

  Stream<WebSocketState> get stateStream => _stateController.stream;
  Stream<Map<String, dynamic>> get eventStream => _eventController.stream;

  Future<void> connect(String namespace) async {
    final token = _storage.token;
    _socket = io.io(
      AppConstants.wsUrl,       // http://192.168.50.174:3000
      io.OptionBuilder()
        .setTransports(['websocket'])
        .setAuth({if (token != null) 'token': token})
        .setPath('/socket.io')
        .build(),
    );

    _socket!
      ..onConnect((_) {
        _reconnectAttempts = 0;
        _setState(WebSocketState.connected);
        _socket!.emit('join', {'namespace': namespace});
      })
      ..onDisconnect((_) => _scheduleReconnect(namespace))
      ..onAny(_processEvent);    // 所有事件 → eventController
  }

  // 指数退避重连：1s → 2s → 4s → 8s，最多 5 次
  int _getReconnectDelay() => switch (_reconnectAttempts) {
    0 => 1,  1 => 2,  2 => 4,  _ => 8,
  };
}
```

## 八、技术要点与风险

### 8.1 关键设计决策

| 决策                      | 说明                                                         |
| :------------------------ | :----------------------------------------------------------- |
| **PageView + 预加载池**   | Feed 流使用 PageView.builder 实现垂直滑动，PlayerPool 维护 3 个控制器实例，预加载前后各 1 个视频，平衡内存与流畅度 |
| **乐观更新 + 失败回滚**   | 购物车增减、选中切换等操作先更新本地状态（即时 UI 反馈），再调 API；失败时回滚到前一状态 |
| **统一 DioClient**        | 所有 API 类复用同一个 DioClient 实例，自动继承 Auth Token 注入、请求日志、指数退避重试、统一异常处理 |
| **WebSocket 事件广播**    | 服务端推送事件通过 eventStream 广播，Provider 层订阅后分发到 State。连接断开自动重连 |
| **本地缓存兜底**          | 用户资料、收藏列表等数据先读本地（Hive/SharedPreferences）保证首屏速度，服务端数据异步覆盖 |

### 8.2 风险评估与应对

| 风险                   | 影响           | 应对措施                                 |
| :--------------------- | :------------- | :--------------------------------------- |
| 后端接口不可用         | 功能不可用     | API 层统一异常处理 + Toast 提示；本地缓存兜底 |
| WebSocket 连接不稳定   | 直播实时数据缺失 | 指数退避重连（最多 5 次）；重连后自动重新加入房间 |
| 视频播放内存溢出       | App 崩溃       | PlayerPool 限制 3 个实例，引用计数淘汰 + 切换页面时 dispose |
| 网络请求并发过大       | 性能下降       | DioClient 内置重试机制；Feed 预加载限制 ±1 索引范围 |
| 乐观更新冲突           | 数据不一致     | API 失败时回滚 state 到前一状态           |

### 8.3 已知局限

| 局限                   | 说明                                         |
| :--------------------- | :------------------------------------------- |
| 点赞仅本地状态         | 直播间点赞未调后端 API，不同步给其他用户     |
| 礼物为 UI 存根         | 礼物面板可展示，但发送仅弹 toast，未走 WebSocket |
| 优惠券领取为存根       | 领取按钮仅弹 SnackBar，未调后端接口          |
| 观众列表为空           | 观众列表 UI 存在，但始终传入空列表           |
| 用户&收藏无服务端同步  | 仅本地 Hive/SharedPreferences 存储，不同步远端 |
| 评论模块未统一         | 视频评论使用独立硬编码地址，未接入 DioClient 的 Auth 拦截器 |

## 十、安全问题

### 已实施的安全措施

| 措施             | 说明                                                         |
| :--------------- | :----------------------------------------------------------- |
| **Token 认证**   | 所有 API 请求通过 DioClient 的 AuthInterceptor 自动注入 `Bearer {token}` |
| **401 自动清理** | 收到 401 响应时自动清除本地存储的 Token，后续请求不再携带无效凭证 |
| **输入校验**     | 评论、聊天消息等用户输入在发送前进行非空和长度校验           |
| **异常统一处理** | 所有网络异常经 DioClient._handleDioException 统一转义为分层异常，避免原始错误信息泄露到 UI |
| **HTTPS 就绪**   | baseUrl 当前使用 HTTP（开发环境），切换为 HTTPS 后 DioClient 无需改动 |

### 待改进的安全项

| 问题                       | 风险等级 | 说明                                       |
| :------------------------- | :------- | :----------------------------------------- |
| 视频评论绕过 Auth 拦截器   | 中       | video_comments_sheet.dart 自建 Dio 实例，未注入 Token |
| 用户 ID 硬编码回退值 `u1`  | 中       | 多处使用 `storage.userId ?? 'u1'`，未登录时以默认用户操作 |
| Token 明文存储             | 低       | SharedPreferences 非加密存储，建议迁移至 FlutterSecureStorage |
| 无请求签名/防篡改          | 低       | 支付类接口缺少请求签名机制                 |
