项目核心目标：实现从“内容浏览 → 商品点击 → 加购/购买 → 订单管理”的完整购物闭环。

技术栈限定：
- 移动端：使用 Flutter（Dart）实现一个跨平台移动应用。
- 后端：使用 Node.js + TypeScript + Express（或 Fastify）提供 REST API 和 WebSocket 服务。
- 数据库：使用 SQLite（方便本地运行），要求表结构包含用户、视频、商品、购物车、订单、评论。

3. 前后端分离：移动端通过 HTTP API 和 WebSocket 与后端通信，严禁后端代码硬编码前端视图。

必须包含的核心功能（用户端）：
- 短视频上下滑列表，每个视频播放器支持播放/暂停、静音、进度条。
- 视频上悬浮商品卡片（图片、标题、价格），点击弹出半屏商品详情（可加购或立即购买）。
- 购物车页面：增删改查商品，勾选结算。
- 订单确认页：收货地址（固定 mock）、商品清单、实付金额，模拟下单。
- 订单列表页：按待支付、待发货、已完成状态筛选。
- 模拟直播间的 WebSocket 实时弹幕、在线人数变化、正在讲解商品状态。

必须包含的核心功能（商家/运营后台）：
由于课题允许后台集成在移动端（通过角色权限），请实现：
- 在同一个 Flutter 应用中，通过登录时的角色（user / merchant）区分界面。
- 商家登录后可见“商家中心”入口，包含：
  * 内容管理：短视频上传/配置（标题、封面、视频 URL、关联商品）。
  * 商品管理：商品列表、添加/编辑/上下架商品。
  * 直播间配置：创建直播间、设置标题、关联商品列表、设置当前讲解商品。
- 商家所有操作均通过后端 API 完成，后端需要校验 `role=merchant`。

后端 API 要求（RESTful）：
- 用户：登录（返回 JWT token 及用户角色）、注册。
- 视频：获取视频列表（分页）、上传视频（商家）、更新视频关联商品。
- 商品：获取商品列表/详情、商家增删改查。
- 购物车：增删改查，合并购物车。
- 订单：创建订单（模拟支付）、获取订单列表（按状态）。
- 评论：发表/获取评论。
- 所有需要权限的接口必须有 JWT 鉴权中间件，商家接口额外检查 role。

数据持久化要求：
- 使用 SQLite（`better-sqlite3` 或 `sqlite3`），提供 schema 初始化脚本和种子数据（至少 3 个视频、5 个商品、1 个直播间）。
- 购物车数据存储在数据库（用户维度）。

实时通信要求：
- 直播间页面使用 socket.io 加入房间，接收 `new_comment`、`online_count`、`current_product` 事件。
- 后端实现简单的直播间模拟：每 3 秒随机变化在线人数，支持发送弹幕。
