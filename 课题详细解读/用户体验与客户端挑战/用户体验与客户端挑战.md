## 角色
你是一名**极端严格的移动端 Flutter 专家**，专注于高并发直播/短视频电商场景。你的任务是：**逐条实现以下用户体验与客户端挑战需求**，任何偏离或遗漏都视为不合格。

## 背景
项目已具备基本直播/视频、商品卡片、购物车、订单等模块。现在需要强化**用户体验、交互规范、性能优化、弱网处理、状态同步、多端适配**。你必须基于现有 Flutter 代码进行改造或新增，确保最终交付版本满足所有条款。

## 需求清单（每一条都必须 100% 满足）

### 一、视频播放体验
- 使用 `video_player` 或 `chewie` 实现视频播放。
- **切换视频时**（比如滑动切换下一个视频）：
  - 前一个视频必须释放资源（`dispose`），避免内存泄漏。
  - 新视频开始播放前，显示占位图或缩略图（不出现白屏）。
  - 新视频的加载进度需要透明反馈（如圆形进度条），从开始加载到可播放的时间应小于 500ms（模拟网络）。
  - 若视频预加载，最多预加载下一个视频的首帧，不可同时加载过多视频导致卡顿。
- **禁止重复加载**：同一个视频在未退出页面时不应重复请求相同的视频数据。

### 二、移动端交互设计（符合直觉）
- **商品浮层卡片**（在视频上）：
  - 支持单指拖动调整位置（可选），但默认位置在右下角且不遮挡核心操作按钮。
  - 点击卡片后弹出**半屏 Modal Bottom Sheet**，高度为屏幕的 70%，圆角 16px。
  - 半屏内可滚动（`DraggableScrollableSheet` 或 `showModalBottomSheet` + `isScrollControlled`），拖拽顶部可调整高度。
- **评论区**：
  - 底部悬浮评论图标，点击弹出**半屏评论面板**，支持新评论输入和滚动历史评论。
  - 评论列表采用懒加载（分页），上拉加载更多。
  - 发送评论时立即乐观更新（先显示自己的评论，失败后回滚并 Toast 提示）。
- **购物车与下单页**：
  - 购物车页面底部固定操作栏（合计金额 + 结算按钮）。
  - 数量修改器（加减按钮）热区不小于 44x44 像素。
  - 下单页收货地址可点击选择（模拟），地址信息完整展示。
  - 提交订单按钮在页面底部固定，滚动时按钮不消失。

### 三、性能优化（列表滚动、加载、缓存）
- **列表懒加载**：
  - 商品列表、视频流、评论列表均支持分页（`page` + `pageSize`）。
  - 滚动到距离底部 2 项时自动加载下一页（`ScrollController` 监听）。
  - 加载时显示底部 loading 指示器，加载失败显示重试按钮。
- **图片缓存**：
  - 商品图、视频封面使用 `cached_network_image` 或类似库，提供占位图和错误占位。
  - 图片内存缓存限制 50MB，磁盘缓存 100MB。
- **避免长列表卡顿**：
  - 视频列表使用 `ListView.builder` 并指定 `addAutomaticKeepAlives: false`。
  - 商品卡片使用 `const` 构造函数减少重建。
- **页面缓存**：切换底部 Tab 时，保留之前的页面状态（使用 `AutomaticKeepAliveClientMixin` 或 `IndexedStack`），不重复请求接口。

### 四、弱网体验与错误处理
- **视频加载失败**：显示提示文案 + 重试按钮，点击重新加载。
- **接口请求失败**：
  - 所有 `http` 请求必须有统一的错误拦截，弹出 SnackBar 或 Toast。
  - 对关键操作（加入购物车、提交订单）在网络失败时，需明确提示“网络异常，请稍后重试”，并提供重试入口。
- **下单失败**：
  - 保留用户已填写的收货地址等信息，不直接清空表单。
  - 弹出对话框提示具体失败原因（库存不足 / 优惠券失效等），返回后可修改。
- **超时处理**：所有 API 请求设置超时 10 秒，超时后按失败处理并提示。

### 五、实时状态同步
- **直播间**：
  - 通过 WebSocket 接收消息类型：`new_comment`、`bullet_screen`、`online_count`、`explaining_product`、`stock_update`。
  - 收到 `explaining_product` 后，高亮对应商品卡片，并展示“主播正在讲解”标签。
  - 收到 `stock_update`（某商品库存变化），立即更新商品卡片的库存状态（如“库存紧张”变“缺货”）。
- **购物车**：
  - 若在商品详情页修改了规格/数量并加入购物车，购物车页面应展示最新数据（可通过刷新或状态管理同步）。
  - 加入购物车后，底部 Tab 栏的购物车图标右上角显示徽章（数量）。
- **订单状态**：支付成功后，订单列表页的对应订单状态自动更新为“待发货”（可以通过 `setState` 重新拉取列表或 WebSocket 推送）。

### 六、多端适配（重点小屏手机）
- 使用 `MediaQuery.of(context).size` 和 `LayoutBuilder` 判断屏幕宽度/高度。
- 小屏手机（宽度 < 360dp）：
  - 商品浮层卡片缩小字体和间距，避免超出屏幕。
  - 底部操作栏高度压缩（但仍保持最小 touch target 44px）。
  - 半屏浮层默认高度为屏幕的 60%，避免键盘弹出时内容被遮挡。
- 使用 `SafeArea` 避开刘海屏和底部虚拟按键。
- 所有按钮点击区域不小于 44×44（逻辑像素），避免误触。
- 对于横屏（如果支持）给出降级提示或者限制为竖屏（`SystemChrome.setPreferredOrientations`）。
