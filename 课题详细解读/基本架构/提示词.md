## 技术栈（严格遵守）

### 移动端（Flutter）

- Flutter 3.22+ / Dart 3.4+（**纯原生 Flutter，禁用 Expo**）
- 纯 Dart 严格模式，**禁用 dynamic**
- 列表：**ListView / PageView + PageController**（实现上下滑全屏流）
- 视频播放：**video_player 2.8+ / chewie 1.8+**
- 状态管理：**Riverpod 2.5+**（或 flutter_riverpod）
- 导航：**go_router 12+**
- 动画：**Flutter 内置 Animation + Hero**
- 手势：**GestureDetector / Dismissible**
- 半屏弹层：**showModalBottomSheet / DraggableScrollableSheet**
- 图片：**cached_network_image 3.3+**
- 网络：**dio 5.4+** + 拦截器
- WebSocket：**socket_io_client 2.0+**
- 存储：**shared_preferences + hive**
- 工程：**analysis_options.yaml 严格规范**

### 后端

- Node.js + TypeScript
- 框架：**Fastify**
- 数据库：**better-sqlite3**
- WebSocket：**socket.io**
- ORM：**Drizzle** 或原生 SQL

### 工程化

- ESLint + Prettier + analysis_options.yaml
- 单 Git 仓库，monorepo 结构
- 目录清晰、分层明确

## 代码规范要求（强制执行）

1. 全 Dart 强类型，**禁止 dynamic**
2. 单个文件不超过 300 行
3. 业务逻辑放入 `service` / `provider`，页面只做 UI
4. 网络请求必须 `try-catch` + 错误处理
5. 图片一律用 `cached_network_image`
6. 列表子组件使用 `const` 构造函数
7. 禁止在 `build` 中重复创建对象
8. 视频播放器必须在 `dispose` 中释放资源
9. 所有异步操作考虑 `mounted` 检查
